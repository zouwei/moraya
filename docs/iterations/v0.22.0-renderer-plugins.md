# v0.22.0 — Renderer Plugin 系统（动态渲染扩展）

**版本：** v0.22.0
**状态：** 计划中
**日期：** 2026-03-03

## 概述

当前 Moraya 的代码块只能渲染 Mermaid 图表（已内置）和语法高亮。随着需求增加，越来越多的"可视化代码块"语言需要支持（ABCjs 五线谱、Vega 数据图、PlantUML 等），但这些库体积各异（50KB～500KB+），全部内置会显著增大安装包。

本次迭代引入 **Renderer Plugin 系统**：开发者维护一份精选白名单注册表，插件 JS 包在用户首次启用时由 Rust 下载到本地并缓存，前端通过 Tauri `asset://` 协议加载（满足 `script-src 'self'` CSP 要求），版本号由 `pnpm version:bump` 自动同步 npm 最新稳定版。**首个 Renderer Plugin：ABCjs 五线谱**。

同时将现有 Plugin System（stdio 插件）与 Renderer Plugin 系统合并为同一个"插件"入口，用户面对统一的管理界面。

---

## 设计原则

| 原则 | 说明 |
|------|------|
| CSP 合规 | 不从外部 CDN 直接加载 JS；Rust 负责下载，前端只读本地缓存 |
| 白名单安全 | 只有 `renderer-registry.ts` 中登记的插件才能被加载，用户不能自行添加外部 JS |
| 版本固定 | 每次发版时 `sync-renderer-plugins.mjs` 自动从 npm 同步最新稳定版到 `renderer-versions.json`，发版间版本冻结 |
| 按需下载 | 插件 JS 只在用户首次启用时下载，不启用不占磁盘 |
| 降级安全 | 插件 JS 尚未下载 / 加载失败时，代码块降级为语法高亮显示原始内容 |
| 统一入口 | 现有 stdio Plugin System 与 Renderer Plugin 共用 SettingsPanel "插件" 标签页 |

---

## 架构总览

```
renderer-registry.ts          ← 开发者维护：插件元数据 + CDN URL 模板
renderer-versions.json        ← 自动维护：每次 version:bump 同步 npm 最新版
        │
        │ 用户启用插件（首次）
        ▼
Rust: invoke('download_renderer_plugin')
  └─ fetch CDN URL
  └─ 保存到 {appDataDir}/renderer-plugins/{name}/{version}/index.js
        │
        │ 代码块渲染时
        ▼
renderer-loader.ts
  └─ convertFileSrc(localPath) → asset://localhost/.../index.js
  └─ import(assetUrl)          ← 满足 CSP script-src 'self'
  └─ 模块缓存（不重复下载/导入）
        │
        ▼
code-block-view.ts (NodeView)
  └─ abcjs 渲染器 → SVG 五线谱
```

### F1.4 加载器（`renderer-loader.ts`）

前端加载、缓存 Renderer Plugin 模块：

```typescript
import { convertFileSrc } from '@tauri-apps/api/core';

// 运行时模块缓存（Map<pluginId, module>）
const moduleCache = new Map<string, unknown>();

export async function loadRendererPlugin(plugin: RendererPlugin): Promise<unknown> {
  if (moduleCache.has(plugin.id)) return moduleCache.get(plugin.id)!;

  const version = getPluginVersion(plugin.npmPackage);
  const localPath = await getLocalPluginPath(plugin.id, version); // invoke check_plugin_cached
  if (!localPath) return null; // 未下载，降级

  const assetUrl = convertFileSrc(localPath);
  // UMD 插件：注入 script 标签，读取 window[exportName]
  // ESM 插件：动态 import(assetUrl)
  const mod = plugin.exportName
    ? await loadUmd(assetUrl, plugin.exportName)
    : await import(/* @vite-ignore */ assetUrl);

  moduleCache.set(plugin.id, mod);
  return mod;
}
```

### F2.2 NodeView 渲染逻辑

**文件**：`src/lib/editor/plugins/code-block-view.ts`

在 `CodeBlockView` 的 `update()` 方法中，当语言为 `abcjs` / `abc` 时：

```typescript
if (isAbcLanguage(lang)) {
  this.renderAbc(content);
  return;
}
```

`renderAbc()` 流程：

1. 调用 `loadRendererPlugin(ABCJS_PLUGIN)` 获取 ABCjs 模块
2. 若返回 null（未下载）→ 降级：显示代码高亮 + 底部提示「启用 ABCjs 插件以渲染五线谱」
3. 若已加载 → `ABCJS.renderAbc(container, content, options)`
4. 渲染容器为 `<div class="abcjs-render">` 替换代码内容区

const ABC_RENDER_OPTIONS = {

## F4: 版本自动化

### F4.1 `sync-renderer-plugins.mjs`（已创建）

脚本已在本次迭代前序工作中创建（`scripts/sync-renderer-plugins.mjs`）。

**工作流**：
1. 读取 `renderer-registry.ts` 中所有 `npmPackage` 字段
2. 查询 `https://registry.npmjs.org/{pkg}/latest` 获取最新稳定版
3. 更新 `renderer-versions.json`

**触发时机**：`pnpm version:bump` 自动调用（已集成到 `bump-version.mjs`）。

### F4.2 GitHub Actions 自动 PR（`.github/workflows/sync-renderer-plugins.yml`）

每周一 UTC 00:00 定时运行，自动检查 npm 是否有新版并创建 PR：

```yaml
name: Sync Renderer Plugin Versions
on:
  schedule:
    - cron: '0 0 * * 1'   # 每周一
  workflow_dispatch:        # 支持手动触发

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 10
      - run: node scripts/sync-renderer-plugins.mjs
      - name: Create PR if versions changed
        uses: peter-evans/create-pull-request@v7
        with:
          commit-message: 'chore: sync renderer plugin versions'
          title: 'chore: sync renderer plugin versions'
          body: |
            Auto-synced renderer plugin versions from npm registry.
            Review the version changes and merge to update the pinned versions.
          branch: chore/sync-renderer-plugins
          delete-branch: true
```

## 验证方案

### Renderer Plugin 架构

1. **白名单安全**：修改 `cdnUrlTemplate` 为非白名单域名，Rust 命令应返回错误
2. **首次启用**：点击「启用」→ 显示下载进度 → 完成后代码块自动渲染
3. **禁用后**：代码块降级为语法高亮，显示 "启用 ABCjs 插件..." 提示
4. **重启后**：已下载插件无需重新下载，直接加载本地缓存
5. **版本更新**：手动触发 `sync-renderer-plugins.mjs` → `renderer-versions.json` 更新 → UI 显示「有新版本」→ 点击更新 → 重新下载新版 JS

### ABCjs 渲染

6. **基础渲染**：输入 ABC 记谱法代码块（语言标识 `abcjs`），WYSIWYG 模式显示五线谱 SVG
7. **`abc` 别名**：语言标识 `abc` 同样触发渲染
8. **语法错误**：ABCjs 解析失败时显示错误提示，不崩溃
9. **Source 模式**：显示原始 ABC 代码，不渲染
10. **主题适配**：Light / Dark 主题下五线谱颜色正确
11. **宽度响应**：窗口缩放时五线谱 SVG 自适应宽度

### 版本自动化

12. **`pnpm version:bump patch`**：输出 "Syncing renderer plugin versions..." + 完成后继续版本升级
13. **无网络时**：`sync-renderer-plugins.mjs` 打印警告并退出 0（非阻断），`version:bump` 继续正常运行
14. **GitHub Actions**：手动触发 workflow → 若有版本变化则创建 PR

### 回归

15. **现有 stdio 插件不受影响**：PluginsPanel（扩展插件）功能与 v0.21.0 一致
16. **Mermaid 渲染不受影响**：Mermaid NodeView 与 Renderer Plugin 架构并列，无冲突
17. **`pnpm check` 0 errors**
