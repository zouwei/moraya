# v0.24.0 — 知识库多层目录 & 图片镜像存储

## 背景

随着知识库内容增多，用户需要通过子目录对文章分类管理。同时 AI 下载图片的存储路径缺乏统一规则，导致图片散落在各处难以维护。本次迭代解决这两个问题，同时新增文件拖拽移动、MCP 全局安装目录规则、MORAYA.md 规则文件置顶与强调色等配套改进。

## F2：列表视图层级化改造

### 当前行为
列表视图显示扁平 `FilePreview[]`，无目录层级，无法体现知识库的文件夹结构。

### 新行为
列表视图改为在 `FileEntry[]` 树上渲染，效果：

```
▾ blog/
      2024年度总结        今年最大的收获是...
      AI趋势分析          大模型正在重塑...
  ▸ drafts/
  文档主页               欢迎使用 Moraya...
```

### 实现细节

- **共用 `expandedDirs`**：树视图和列表视图共享同一折叠状态，切换视图时保持展开状态
- **文件预览查找**：维护 `previewMap = Map<path, FilePreview>`，渲染文件行时按路径查找摘要
- **缩进规则**：`padding-inline-start: {0.75 + depth * 1}rem`（与树视图一致）
- **过滤规则**：渲染时跳过 `name === 'images'` 的目录及其子树

### 实际实现细节

- **数据源**：列表视图遍历 `filteredTree`（`FileEntry[]` 树），而非原先的扁平 `filteredPreviews`
- **目录行**：整行宽度显示，行高与文件行一致（`min-height: 36px`），方便拖放目标定位
- **字号调整**：
  - `.list-item-title`：`var(--font-size-xs)`（12px）
  - `.list-item-preview`：`11px`
  - `.list-dir-name`：`var(--font-size-xs)`, `font-weight: 500`, `color: var(--text-secondary)`
- **data 属性**：每个目录 button 携带 `data-folder-path`，每个文件 button 携带 `data-file-path`，供拖拽命中检测使用

### 列表视图 Snippet（伪代码）
```svelte
{#snippet listItem(entry: FileEntry, depth: number)}
  {#if entry.is_dir}
    <!-- 目录行：整行宽度，高度与文件行一致 -->
    <button class="list-dir-item" style="padding-inline-start:{0.75+depth}rem"
      data-folder-path={entry.path}
      onclick={() => toggleDir(entry.path)}>
      <span class="tree-icon" class:expanded={expandedDirs.has(entry.path)}>...</span>
      <span class="list-dir-name">{entry.name}</span>
    </button>
    {#if expandedDirs.has(entry.path) && entry.children}
      {#each entry.children.filter(c => !isReservedDir(c)) as child}
        {@render listItem(child, depth + 1)}
      {/each}
    {/if}
  {:else}
    {@const preview = previewMap.get(entry.path)}
    <button class="list-item" style="padding-inline-start:{0.75+depth+1}rem"
      data-file-path={entry.path}
      onmousedown={(e) => startFileDrag(e, entry)}
      onclick={() => onFileSelect(entry.path)}
      oncontextmenu={...}>
      <span class="list-item-title">{getDisplayName(entry.name)}</span>
      {#if preview?.preview}
        <span class="list-item-preview">{preview.preview}</span>
      {/if}
    </button>
  {/if}
{/snippet}
```

## F4：AI 图片镜像存储规则

### 目录结构约定

```
{kbRoot}/
├── blog/
│   └── 2024/
│       └── my-article.md
└── images/                    ← 保留目录，侧边栏不显示
    ├── temp/                  ← 文章未保存时的临时目录
    └── blog/
        └── 2024/
            └── my-article/   ← 与文章同名（去掉 .md）
                └── hero.jpg
```

### 路径计算规则（最大 3 层）

```typescript
// src/lib/services/ai/image-path-utils.ts

export function computeImageDir(
  articlePath: string | null,
  kbRoot: string
): string {
  const base = `${kbRoot}/images`;

  // 文章未保存 → 临时目录
  if (!articlePath) return `${base}/temp`;

  // 计算相对路径（去掉 kbRoot 前缀和 .md 后缀）
  const rel = articlePath
    .replace(kbRoot + '/', '')
    .replace(/\.md$/i, '');

  // 按 / 拆分路径段
  const parts = rel.split('/');

  // 最大 3 层：最多保留前 2 个子目录 + 文章名
  // parts = ['blog', '2024', 'my-article'] → 取全部（3段）
  // parts = ['a', 'b', 'c', 'd', 'article'] → 取 ['a', 'b', 'article']（截断中间）
  const MAX_DEPTH = 3;
  let dirParts: string[];
  if (parts.length <= MAX_DEPTH) {
    dirParts = parts;
  } else {
    // 保留前 MAX_DEPTH-1 个目录 + 最后一个（文章名）
    dirParts = [...parts.slice(0, MAX_DEPTH - 1), parts[parts.length - 1]];
  }

  return `${base}/${dirParts.join('/')}`;
}
```

### F4d：保存后迁移（unsaved → correct path）

**触发时机**：文件首次命名保存（`oldPath === null && newPath !== null`）

**迁移函数** `migrateTempImages(newFilePath, kbRoot)` 位于 `file-service.ts`：

```typescript
export async function migrateTempImages(
  newFilePath: string,
  kbRoot: string
): Promise<{ moved: number; updatedPaths: Record<string, string> }> {
  const tempDir = `${kbRoot}/images/temp`;
  const targetDir = computeImageDir(newFilePath, kbRoot);

  // 1. 列出 tempDir 中的图片文件
  // 2. 对每个文件：
  //    a. mkdir -p targetDir
  //    b. 读取文件内容 → 写入到 targetDir/filename
  //    c. 删除 tempDir/filename（保留 temp/ 空目录）
  // 3. 返回 { 迁移数量, oldPath→newPath 映射 }
}
```

**调用位置**（`+page.svelte` 保存逻辑中）：

```typescript
// 文件首次保存时
if (!previousPath && currentFilePath) {
  const kb = filesStore.getActiveKnowledgeBase();
  if (kb && currentFilePath.startsWith(kb.path)) {
    const result = await migrateTempImages(currentFilePath, kb.path);
    if (result.moved > 0) {
      // 替换编辑器内容中的 images/temp/filename → images/.../filename
      let content = getMarkdown(editorView);
      for (const [oldRel, newRel] of Object.entries(result.updatedPaths)) {
        content = content.replaceAll(oldRel, newRel);
      }
      // 更新编辑器并重新保存
      editorView.syncContent(content);
      await saveFile(currentFilePath, content);
    }
  }
}
```

## F5：侧边栏文件拖拽移动

### 背景

WKWebView（Tauri on macOS）的 HTML5 Drag and Drop API 在 `<button>` 元素上行为不可靠：`ondragstart` 在按钮上无法稳定触发，`ondragover` 事件在 button 之间切换时容易丢失。因此放弃 HTML5 DnD，改用鼠标事件 + `document.elementFromPoint()` 实现。

### 交互设计

- 在文件行（`.list-item`）按住左键并移动 ≥5px 触发拖拽
- 拖拽时显示半透明幽灵跟随鼠标
- 鼠标悬停目标文件夹时高亮（`drop-target` 类）
- 悬停在侧边栏内容区域（无文件夹命中）则高亮整个侧边栏边框（`drop-root` 类）表示放入 KB 根目录
- 松手触发移动，调用 `invoke('rename_file', ...)` 并刷新文件树

### 核心实现

**`startFileDrag(event, entry)`**：
```typescript
function startFileDrag(event: MouseEvent, entry: FileEntry) {
  if (event.button !== 0 || entry.is_dir) return;
  const startX = event.clientX, startY = event.clientY;
  let started = false;
  // 注意：使用 plain let 变量 _dragPath，避免 $state 响应式读取延迟
  function onMove(e: MouseEvent) {
    if (!started) {
      if (Math.hypot(e.clientX - startX, e.clientY - startY) < 5) return;
      started = true;
      _dragPath = entry.path;
      draggedFilePath = entry.path; // $state，用于视觉反馈
      createDragGhost(entry.name, e.clientX, e.clientY);
    }
    moveDragGhost(e.clientX, e.clientY);
    dropTargetPath = findFolderAtPoint(e.clientX, e.clientY);
  }
  async function onUp(e: MouseEvent) {
    // cleanup → 移动文件 → 刷新树
  }
  document.addEventListener('mousemove', onMove);
  document.addEventListener('mouseup', onUp);
}
```

**`findFolderAtPoint(x, y)`**（三级命中检测）：
```typescript
function findFolderAtPoint(x: number, y: number): string | null {
  // 临时隐藏幽灵以避免命中自身
  if (_dragGhost) _dragGhost.style.display = 'none';
  const el = document.elementFromPoint(x, y);
  if (_dragGhost) _dragGhost.style.display = '';

  // 1. 命中文件夹 button → 返回该文件夹路径
  const folderBtn = el?.closest('[data-folder-path]') as HTMLElement | null;
  if (folderBtn?.dataset.folderPath) { ... return target; }

  // 2. 命中文件 button → 返回文件所在目录
  const fileBtn = el?.closest('[data-file-path]') as HTMLElement | null;
  if (fileBtn?.dataset.filePath) { return parentDir; }

  // 3. 命中侧边栏内容区域 → 返回 KB 根目录
  if (el?.closest('.sidebar-content') && folderPath) { return folderPath; }

  return null;
}
```

## F6：MCP 全局安装目录

### 设计原则

通过 AI 安装的 MCP 服务（动态服务）属于用户级全局资源，应安装在用户主目录下的隐藏目录，而非应用私有数据目录，以便跨应用访问和手动管理。

### 目录规则

```
~/.moraya/mcp/
├── runtime/   ← 临时运行时服务（重启不保留）
├── temp/      ← 临时服务
└── saved/     ← 已保存的持久服务
```

### 实现

**`src/lib/services/mcp/container-manager.ts`**：
```typescript
import { homeDir } from '@tauri-apps/api/path';

let cachedHomeDir: string | null = null;
async function getServicesDir(): Promise<string> {
  if (!cachedHomeDir) cachedHomeDir = await homeDir();
  const home = cachedHomeDir.replace(/[/\\]$/, '');
  return `${home}/.moraya/mcp`;
}
```

**`src-tauri/capabilities/default.json`** 新增权限：
```json
{ "identifier": "fs:scope", "allow": [{ "path": "$HOME/.moraya/**" }] }
```

## F7：MORAYA.md 规则文件置顶与强调色

### 设计

- `MORAYA.md` 是知识库的 AI 规则文件，需在侧边栏中始终置于第一位，不受排序方式影响
- 文件名显示为强调色（`var(--accent-color)`，与 AI 面板发送按钮同色），直观区分规则文件
- 预览摘要显示为强调色渐灰（`color-mix(in srgb, var(--accent-color) 45%, var(--text-muted))`）

### 置顶实现

```typescript
function pinMorayaMd(entries: FileEntry[]): FileEntry[] {
  const idx = entries.findIndex(e => !e.is_dir && e.name === 'MORAYA.md');
  if (idx <= 0) return entries;
  const result = [...entries];
  result.splice(idx, 1);
  result.unshift(entries[idx]);
  return result;
}

// filteredTree 在过滤后应用置顶
let filteredTree = $derived(pinMorayaMd(filterTree(filterReserved(fileTree), searchQuery)));
```

> `pinMorayaMd` 仅作用于根级条目，子目录内同名文件不受影响。

### 强调色 CSS

```css

  color: var(--accent-color);
```

> 注意 CSS 优先级：使用复合选择器（`.list-item-title.moraya-rule`，specificity 0,2,0）确保覆盖同文件中的单类声明（`.list-item-title`，specificity 0,1,0）。
