# v0.25.0 — 正则搜索替换 & Base64 图片支持

## 概述

本次迭代为 Moraya 编辑器的「查找与替换」功能进行全面升级，新增正则表达式搜索、跨节点多行搜索替换能力，并优化搜索栏布局使其适应全宽窗口。同时验证并完善 Base64 内嵌图片的显示支持。

---

## F1：正则表达式搜索与替换

### 需求描述

在搜索栏中新增「正则模式」切换按钮（`.*`），开启后搜索输入框内容作为 JavaScript 正则表达式解析。

### 正则语法（JavaScript RegExp / ECMAScript 规范）

| 类别 | 语法 | 说明 |
|------|------|------|
| 基本元字符 | `.` `*` `+` `?` `\|` | 任意字符、重复、可选、或 |
| 分组 | `(...)` `(?:...)` | 捕获组、非捕获组 |
| 字符类 | `[abc]` `[a-z]` `[^abc]` | 字符集、范围、取反 |
| 量词 | `{n}` `{n,}` `{n,m}` | 精确/最少/范围重复 |
| 预定义字符类 | `\d` `\w` `\s` `\b` | 数字、单词、空白、边界 |
| 否定字符类 | `\D` `\W` `\S` `\B` | 对应否定形式 |
| 前瞻/后瞻 | `(?=...)` `(?!...)` `(?<=...)` `(?<!...)` | 正向/负向 前瞻/后瞻 |
| Unicode | `\u{XXXX}` | Unicode 码点（需 `u` flag） |
| 换行匹配 | `\n` | 匹配跨节点/跨行边界 |

### 替换语法（捕获组引用）

| 语法 | 说明 |
|------|------|
| `$1` `$2` ... `$9` | 引用第 N 个捕获组 |
| `$&` | 引用整个匹配 |
| `` $` `` | 匹配之前的文本 |
| `$'` | 匹配之后的文本 |
| `$$` | 转义为字面量 `$` |

### 交互设计

- 搜索栏右侧新增 `.*` 按钮，点击切换正则模式（高亮 = 已启用）
- 正则语法错误时，匹配计数位置显示红色错误提示
- 正则模式下 `\n` 可匹配跨段落/跨块换行
- 非正则模式下，多行输入中的换行符也能匹配文档中的实际换行

### 实现要点

**ProseMirror（Visual 模式）**：

- 构建文档的平坦文本表示（flat text），块级节点边界插入 `\n`
- 维护 `offsetMap: { textOffset → pmPos }[]` 映射关系
- 正则/文本搜索均在 flat text 上执行，结果通过 offsetMap 转换回 ProseMirror 位置
- 替换操作：正则模式使用 `String.prototype.replace()` 处理捕获组引用；跨节点替换需要 `tr.replaceRange()` 而非 `tr.replaceWith(schema.text())`

**SourceEditor（Source 模式）**：

- 直接在 `content` 字符串上执行 `RegExp` 匹配
- 替换使用 `content.replace(regex, replaceStr)` 原生支持捕获组

### 错误处理

- `new RegExp(pattern, flags)` 包裹 try-catch
- 无效正则：不执行搜索，显示 `search.regexError` 错误提示
- 超时保护：匹配结果超过 10,000 个时截断，避免 ReDoS

## F2：多行搜索与替换（Sublime 风格）

### 需求描述

搜索和替换输入框支持多行内容，参考 Sublime Text 的交互方式。

### 交互设计

- 输入框改为 `<textarea>`，默认 1 行高度
- **Alt+Enter**（macOS/Windows/Linux）：在输入框中插入换行
- **Cmd+Enter / Ctrl+Enter**：在输入框中插入换行（备选快捷键）
- **Enter**：仍然触发「查找下一个」（搜索框）或「替换」（替换框）
- **Shift+Enter**：查找上一个
- 输入框高度随内容自动增长，最多 6 行，超出后滚动
- 换行在输入框中可见为实际的多行文本

### 实现要点

- `<input type="text">` → `<textarea rows={动态计算}>`
- `rows = Math.min(Math.max(lineCount, 1), 6)`
- `resize: none; overflow-y: auto; max-height: 8rem`

## F3：搜索栏布局优化

### 需求描述

搜索栏铺满编辑器全宽，输入框自适应拉伸，按钮靠右对齐。

### 布局结构

```
┌─[▶]─┬──────────────────────────────────┬─────────────────────────┐
│     │  Find textarea (flex: 1)         │ 1/5  [.*] [Aa] [↑][↓][×]│
│     ├──────────────────────────────────┼─────────────────────────┤
│     │  Replace textarea (flex: 1)      │ [Replace] [Replace All] │
└─────┴──────────────────────────────────┴─────────────────────────┘
```

### 变更

- 移除 `.search-fields { max-width: 500px }` 限制
- 输入框 `flex: 1` 自适应剩余宽度
- 按钮区域 `flex-shrink: 0` 固定在右侧
- 搜索行：输入框 + 右侧（匹配计数 + 按钮组）
- 替换行：输入框 + 右侧（替换 + 全部替换按钮）

## F4：Base64 内嵌图片支持

### 需求描述

支持在 Markdown 中直接使用 Base64 编码的图片，适用于小图标、签名等小体积图片场景。

### Markdown 语法

```markdown
![alt text](data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAUA...)
```

### 当前状态

| 检查项 | 状态 | 说明 |
|--------|------|------|
| CSP `img-src` | ✅ 已包含 `data:` | `tauri.conf.json` |
| Schema `image.src` | ✅ 无协议限制 | `schema.ts` |
| markdown-it 解析 | ✅ 原生支持 `data:` URL | — |
| ProseMirror 渲染 | ✅ `<img src="data:...">` 正常 | — |
| 序列化回 Markdown | ✅ 直接输出 `src` | `markdown.ts` |

### 需要验证/修复

1. **图片粘贴**：`editor-props-plugin.ts` 的 paste handler 当前可能将粘贴图片保存为本地文件。需确认是否保留 base64 粘贴路径的选项
2. **图片工具栏**：ImageToolbar / ImageContextMenu 的「另存为」「上传到图床」操作需兼容 `data:` 前缀的 src
3. **安全过滤**：拒绝 `data:text/html`、`data:application/javascript` 等危险 MIME 类型，仅允许 `data:image/*`
4. **文件体积提示**：当 base64 图片 > 100KB 时，在状态栏或工具栏提示「建议使用外部图片文件」

## 验证方案

### F1 正则搜索

1. 开启正则模式，输入 `\d+` → 匹配所有数字序列
2. 输入 `(foo)(bar)` 搜索，替换为 `$2$1` → 验证捕获组交换
3. 输入无效正则 `[unclosed` → 显示红色错误提示，不崩溃
4. 输入 `(?<=# ).*` → 匹配所有标题文本（后瞻）

### F2 多行搜索

5. 在搜索框按 Alt+Enter 插入换行，输入多行文本 → 验证跨段落匹配
6. 正则模式输入 `第一行\n第二行` → 匹配跨段落内容
7. 替换框也支持多行，替换结果包含换行

### F3 布局

8. 窗口宽度 800px~1920px → 搜索栏输入框自适应拉伸，按钮始终靠右
9. 多行输入时搜索栏高度自动增长，最多 6 行

### F4 Base64 图片

10. 粘贴 `![](data:image/png;base64,iVBOR...)` → 编辑器内正常显示图片
11. 右键 base64 图片 →「另存为」功能正常
12. 输入 `![](data:text/html;base64,...)` → 图片不渲染（安全过滤）
