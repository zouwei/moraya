# v0.23.0 — AI 输入交互升级与 Voice 声源切换

**版本：** v0.23.0  
**状态：** 已完成
**日期：** 2026-03-06

## 概述

本次迭代统一处理三条需求线：

1. AI 输入框交互改版（参考 Cursor/Codex 输入体验，按钮内嵌输入框）。  
2. Voice 能力升级（输入框内语音转文字 + Voice 页面改版 + 声源切换前置能力）。
3. Chat 端到端语音对话模式（条件显示波浪按钮 + 通话中停止按钮 + Chat 设置页新增端到端语音 AI 配置）。

本文件只定义方案、范围、验收与风险，待文档确认后再进入开发。

## 目标与范围

### 目标

- 完成 AI 输入区三层内嵌式交互升级。
- 新增输入框内语音转文字交互（录音态按钮层切换）。
- 新增端到端语音 AI 对话入口与通话中交互（文本/图片补充不被阻断）。
- 完成 Chat 设置页改版：区分“会话 AI”与“端到端语音 AI”（多配置）。
- 改造原 Voice 页面关键视觉与操作布局。
- 将“声源切换（麦克风/系统声源/双声源）”纳入本版本前置能力设计，确保后续需求可持续迭代。
- 同一供应商多节点 URL（如阿里云北京/新加坡）统一适配，保证结构一致时零解析改动。

### 非目标

- 本阶段不调整 STT 供应商解析协议（`speech_proxy.rs` 解析逻辑不做功能扩展）。
- 本阶段不新增离线 ASR 引擎。
- 本阶段不引入复杂音频后处理（AEC 高级回声抵消、分轨后端重建等）。

## 前置能力：声源切换可行性与设计

### 可行性结论

- `麦克风输入`：已具备，当前 `getUserMedia` + `AudioWorklet` 链路可复用。
- `系统声源输入`：技术上可实现，但依赖平台能力与权限（需做运行时能力探测）。
- `双声源同时输入`：可实现，采用 Web Audio 将两个输入流混合后统一送入现有 PCM 管线。

结论：需求可做，建议拆为独立前置子阶段（F1），完成后再叠加输入框语音交互。

### 设计要点

- 新增声源模式：
  - `mic`（默认）
  - `system`
  - `mixed`（mic + system）
- 模式约束：
  - `面试模式` 强制使用 `system`，并关闭 `mic`。
- 新增能力探测：
  - 是否可获取系统音轨。
  - 是否允许双流并发。
  - 失败时回退到 `mic` 并给出明确提示。
- 音频链路统一：
  - 所有模式最终输出为单路 `mono + 16kHz + PCM16`，复用现有 `speech_proxy_send_audio`。
- 双声源混音：
  - `MediaStreamSource(mic)` + `MediaStreamSource(system)` → `GainNode`（各自增益）→ `ChannelMerger/MediaStreamDestination` → PCM Worklet。
- 录音生命周期：
  - 开始录音时锁定声源模式，录音中不允许切换（避免流重建导致断句/丢包）。

### 同厂商多节点适配（URL 变、结构不变）

- Provider 解析器按 `provider` 维度绑定，不按 `baseUrl` 绑定。
- 在配置层引入“节点预设”，用户可在同一 provider 下切换 URL。
- 示例（阿里云 DashScope）：
  - 北京：`wss://dashscope.aliyuncs.com/api-ws/v1/inference`
  - 新加坡：`wss://dashscope-intl.aliyuncs.com/api-ws/v1/inference`
- 要求：节点切换不触发解析分支变化，不改 `SpeechEvent` 数据结构。

## 需求 1：AI 输入框改版（Cursor/Codex 风格）

### 布局规范

输入框整体改为“单容器三层结构”：

1. 顶层：附件行（图片预览、附件状态）。
2. 中层：文本输入区（多行弹性）。
3. 底层：操作按钮行。

所有按钮都包入同一输入容器内部。

### 交互规则

- 若无附件：文本输入区自动贴顶（不保留附件占位空白）。
- 输入框自适应高度上限从 `5` 行提升为 `7` 行。
- 超过 `7` 行后：文本区内部滚动，不再继续拉高容器。
- 底栏按钮：
  - 左侧：`[+] [📎]`
  - 右侧：`[录音 SVG] [主动作 SVG]`
- `+` 菜单向上弹出，保留后续扩展入口。

### 主动作按钮条件规则（新增）

| 条件 | 右侧主动作按钮图标 | 点击行为 |
|------|--------------------|----------|
| 仅配置“会话 AI”，未配置“端到端语音 AI” | `send.svg` | 发送文本/图片消息 |
| 已配置“端到端语音 AI” 且输入框无文本/附件 且未在通话中 | `wave.svg` | 启动端到端语音对话 |
| 已配置“端到端语音 AI” 且输入框有文本或附件 且未在通话中 | `send.svg` | 发送文本/图片消息 |
| 端到端语音对话进行中 | `stop.svg`（独立按钮） | 停止端到端语音对话 |

说明：
- 你提出的规则已固化：`send` 位置在“空输入 + 已配置端到端语音 AI”时显示 `wave.svg`。
- 一旦用户输入任何文字或附加图片，主动作自动切回 `send.svg`。

### 样式约束

- 保持现有主题变量体系，不引入新的视觉主题。
- 移动端与窄宽度下按钮不换行，必要时压缩间距和文字。

## 需求 2：输入框内语音转文字

### 状态机

- `idle`：显示 `[录音 SVG] [主动作 SVG（send/wave 条件切换）]`
- `recording`：右侧切换为 `（波形动画） [X] [✓]`
- `cancelled`：丢弃本次语音，恢复 `idle`
- `committed`：将识别文本回填输入框，恢复 `idle`

### 行为定义

- 点击 `录音`：开始语音采集与实时识别。
- 点击 `X`：终止并丢弃当前会话文本，不改输入框已有内容。
- 点击 `✓`：将当前识别结果填入输入框（建议“插入到光标处”，无光标时追加到末尾）。
- 提交 `✓` 后仅回填，不自动发送；由用户点击 `发送` 决定提交。
- 配置缺失时点击 `录音`：提示并可跳转语音设置。

### 兼容约束

- 与现有 `Ctrl/Cmd + Enter` 发送快捷键兼容。
- 录音过程中 `发送`按钮不可见，避免并发提交冲突。

## 需求 3：端到端语音 AI 对话模式（新增）

### 入口与显示条件

- 主动作按钮（发送位）是否显示 `wave.svg`，由 Chat 设置中的“端到端语音 AI 配置”决定。
- 仅当存在可用端到端语音 AI 配置时，空输入状态显示 `wave.svg`。
- 当用户输入任意文本或添加图片时，主动作按钮立即切为 `send.svg`。

### 会话态交互

- 点击 `wave.svg` 后，进入端到端语音 AI 对话模式。
- 底部切换为语音对话控制布局，输入区保持可编辑。
- 语音对话进行中：
  - 隐藏 `+` 与 `录音转写按钮`。
  - 保留 `附件上传` 与 `文字输入`，支持中途补充文字/图片。
  - 右侧新增独立 `stop.svg`（停止语音对话）按钮。
  - 文本 `send.svg` 默认隐藏；当存在待发送文本/图片时显示，避免与 `stop.svg` 冲突。

### 冲突规避规则（你提出的关键点）

- `stop.svg` 与 `send.svg` 不复用同一状态按钮，而是并列策略：
  - `stop.svg`：语音对话期间始终可见。
  - `send.svg`：仅在用户有待发送文本/图片时显示。
- 这样既能随时停止语音通话，也不会阻断中途补充文本/图片发送。

### 状态机（端到端语音）

- `text_idle`：常规输入态（主动作 `send` 或 `wave`）
- `e2e_connecting`：端到端语音会话建立中
- `e2e_active_no_draft`：会话中，无补充文本/图片（仅 `stop`）
- `e2e_active_with_draft`：会话中，有补充文本/图片（`send + stop`）
- `e2e_stopping`：会话结束中，收尾并恢复常规输入态

## 需求 5：Voice 页面改版

### UI 调整

- `Ready/Recording/Paused...` 状态标签移动到语音面板左上角。
- 新增 `声源设置` 下拉：
  - `转录模式` 下可选 `mic` / `system` / `mixed`。
  - `面试模式` 下强制 `system`，并关闭 `mic` 采集。
- 新增 `语音模式` 下拉：`转录模式（默认）`、`面试模式`。
- 录音主按钮放入语音面板内部，使用大号麦克风 SVG 图标作为主操作入口。
- 底部控制区保留辅助动作（返回、转为文档/摘要），但主录音动作以面板内大按钮为核心。

### 交互规则

- 录制开始后允许切换 `语音模式`（转录/面试），模式切换即时生效。
- `转录模式`下按用户设置采集声源（mic/system/mixed）。
- `转录模式（默认）`：
  - 正常实时转录。
  - 点击 `转为文档` 时，将转录内容**追加到当前打开文档末尾**（不新建文档）。
- `面试模式`：
  - 进入模式时自动切换声源为 `system`，并关闭 `mic`。
  - 正常进行实时转录（语音内容保持左侧输出）。
  - 当录入静默停顿超过 `3s`，触发一次 AI 回答请求。
  - AI 只接收“从上次传入结束位置之后”的新增转录文本（增量传入）。
  - AI 答案显示在对应转录段下方，且靠右输出，形成对话样式。

### 面试模式输出规范（新增）

- `左侧`：语音转录流（候选人/面试官原话）。
- `右侧`：AI 回答流（基于最近增量上下文生成）。
- 触发策略（文档阶段）：
  - 触发条件：`静默 > 3 秒`。
  - 传入内容：`[lastForwardedEnd, currentEnd]` 区间内新增文本（delta）。
  - 游标更新：AI 请求成功或失败后，都推进到本次提交末尾，避免重复提交同一段。
  - 失败降级：AI 请求失败时右侧显示错误气泡，不影响左侧转录。

## 验收标准（Definition of Done）

- AI 输入区满足三层内嵌结构，视觉上所有按钮均在输入容器内。
- 输入框最多展示 7 行，超限后仅内部滚动。
- 录音、发送、波浪、停止按钮均为 SVG 图标。
- 主动作按钮满足条件切换规则：
  - 无端到端配置或有输入内容时显示 `send.svg`
  - 端到端可用且空输入时显示 `wave.svg`
- 录音按钮点击后右侧按钮层切换为 `波形 + X + ✓`，交互路径完整。
- `✓` 可把语音结果回填输入框，`X` 可取消且不污染输入内容。
- 端到端语音通话中 `stop.svg` 始终可见；有补充文本/图片时额外出现 `send.svg`。
- Chat 设置页存在“会话 AI / 端到端语音 AI”两个配置分区，并支持多配置。
- Chat 设置页可创建并保存 7 家端到端 provider 配置项（至少通过配置层与持久化层验收）。
- Voice 页面状态标签位于左上角，存在大号麦克风主按钮。
- 转录模式声源可选 `mic/system/mixed`，不可用场景有明确回退提示。
- Voice 页面支持 `转录模式` 与 `面试模式`，且录制后可切换模式。
- 面试模式下强制 `system` 声源并关闭 `mic`，且声源控件锁定为只读显示。
- `转为文档`会将内容追加到当前打开文档末尾。
- 面试模式下：语音转录靠左、AI回答靠右，且可按问题分段输出。
- 面试模式 AI 触发规则为：录入静默停顿 `>3s` 触发一次，并只传入上次结束游标后的新增文本（delta）。
- 同一 provider 不同 URL 节点切换后，转录输出结构一致、页面正常展示。

## 风险与回退策略

- 系统音采集受平台/权限影响：必须做运行时探测；转录模式可回退 `mic`，面试模式不可回退（应阻止进入并提示）。
- 双声源混音可能导致识别准确率下降：提供默认增益配置与后续可调参数入口。
- 输入区重构与模板面板/附件条并存有布局冲突风险：需要响应式快照回归测试。
- 端到端语音通话与文本补充并行时存在状态冲突风险：以“`stop` 独立、`send` 条件显示”作为唯一策略收敛交互。
- 面试模式基于“静默>3s”触发，可能在自然停顿处过早调用 AI：需加入最小增量长度阈值（如 10 字）与失败气泡提示。
- 面试模式强制 system 声源，若系统音不可采集则应提示并阻止进入面试模式。
